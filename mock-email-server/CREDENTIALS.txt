# Test Credentials for Mock Email Server

## Default Test Account

**Email Address:** test@example.com
**Password:** password123

**IMAP Configuration:**
- Host: localhost
- Port: 1143
- Security: None (Plain/No SSL)

**SMTP Configuration:**
- Host: localhost
- Port: 1587
- Security: None (Plain/No SSL)

---

## Notes

- The mock server accepts ANY password for testing purposes
- You can use any email/password combination and it will work
- No SSL/TLS is required (use plain/unencrypted connections)
- The server runs locally on your machine

## Quick Copy-Paste Format

```
Email: test@example.com
Password: password123
IMAP Host: localhost
IMAP Port: 1143
SMTP Host: localhost
SMTP Port: 1587
```

## For Frame Email Client Login Form

When logging into Frame Email Client, enter:

```json
{
  "email": "test@example.com",
  "password": "password123",
  "imap_host": "localhost",
  "imap_port": 1143,
  "smtp_host": "localhost",
  "smtp_port": 1587
}
```

## Additional Test Users (Optional)

You can add more test users by editing `test_data.json`:

```json
{
  "email": "alice@example.com",
  "password": "alice123",
  "name": "Alice Smith"
},
{
  "email": "bob@example.com",
  "password": "bob123",
  "name": "Bob Johnson"
}
```

## Environment Variables (Optional)

If you want to use environment variables:

```bash
export EMAIL_USER="test@example.com"
export EMAIL_PASS="password123"
export IMAP_HOST="localhost"
export IMAP_PORT="1143"
export SMTP_HOST="localhost"
export SMTP_PORT="1587"
```

## Connection String Format

IMAP: `imap://test@example.com:password123@localhost:1143`
SMTP: `smtp://test@example.com:password123@localhost:1587`

---

**Security Note:** These are test credentials for a local mock server only. 
Never use these credentials for real email accounts or production systems.
